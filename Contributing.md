# Contributing

Thanks for your interest in contributing to My Prose App.

## How to contribute

- Open an issue for bugs, ideas, or improvements.
- Keep changes small and focused.
- Follow the existing Java style in the project.
- Write clear, readable code.
- Add or update documentation if needed.

## Coding style

- Use descriptive class and method names.
- Keep methods short and simple.
- Format code consistently.
- Prefer clean, easy-to-read solutions over clever ones.

## Pull request process

1. Fork the repository.
2. Create a feature branch.
3. Make your changes.
4. Test the project locally.
5. Open a pull request with a short description of the change.

## Commit messages

Use short, meaningful commit messages, for example:

- `Add sentence start implementation`
- `Fix output formatting`
- `Update README`

## Questions

If something is unclear, open an issue before starting larger changes.
